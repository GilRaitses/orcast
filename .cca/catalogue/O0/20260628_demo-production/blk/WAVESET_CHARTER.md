# Stage 1, BLK (blocking rehearsal), waveset charter

Family DBLK. Parent: `../PROGRAM.md`. Beats: `../BEAT_SET.md`. Enters only after SET
confirms `READINESS.md`.

## Intent

Clear each live beat's moving parts, alone then together, in the running app. No capture.
A beat advances to CAM only when every one of its moving parts is CLEARED.

## Unit of parallelism

One First AD per live beat (A1, A2, A3, A4, A5, A6, B1, B2, B3). Slide beats (C1, C2) skip
BLK. Disjoint scope: a First AD touches only its beat's surfaces.

## Method (per First AD)

1. Read the beat record (Say / Show / Evidence / Honesty) in `../BEAT_SET.md`.
2. Decompose the Show into ordered moving parts.
3. Clear each part in isolation, citing the built file and symbol and a Read-examined
   runtime observation (the real rendered pane, the real API turn).
4. Clear the parts together in the running app, in beat order.
5. Confirm the Honesty caption is actually presentable on screen for this beat (the real
   figure, the gate caveat, the modeled/estimate label).
6. Write the blocking card: each part CLEARED or BLOCKED with a numbered defect, file, and
   symbol.

## Moving-parts seed (director refines per beat)

- A1: forecast map mounts; confidence meter shows the real 0% promoted; hero copy correct.
- A2: public console accepts the prompt; planning panels render; CXR redaction holds.
- A3: trip-branch control present; selecting a branch routes and re-grounds.
- A4: map-cell provenance modal opens with kernels + verdicts; `/gates` integrity list +
  PSTH render.
- A5: `/ask` pin + question; Check sighting returns a Bedrock-badged grounded reply.
- A6: `/journal` save + publish; `/moderation` approve writes to the queue.
- B1: reviewer console plan turn renders the orchestrator trace.
- B2: hydrophone-detection annotation surface + provenance lineage render.
- B3: twin sandbox renders; land meets the waterline (W2.6); resting framing is tight
  (W-PERFUX-BUILD); any choreographed pan present is non-random.

## Output

- `findings/BLK-<beat>.md` per beat with each part CLEARED / BLOCKED.
- `BLK-ROLLUP.md`: the beat-by-beat clear state; which beats advance to CAM.

## Advance gate

A beat advances to CAM only when every moving part is CLEARED. A BLOCKED part is a numbered
defect returned to the build owner (First ADs do not fix product code).

## Escalation

First AD to Director on any ambiguous clear; Director resolves or escalates to O0.
