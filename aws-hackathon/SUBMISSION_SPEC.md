# AWS Hackathon — Submission Spec

## What to Submit

### 1. Text Description
- Tell us **which one of the AWS Databases** you used.

### 2. Demo Video (< 3 minutes, YouTube preferred)
- Explain **what problem** your application is solving, **for whom**, and **why** you chose this problem.
- Include footage that shows your **working application**.
- Explain the **AWS Databases** used in the submission.

### 3. Vercel Links
- Published **Vercel Project Link**.
- **Vercel Team ID**.

### 4. Architecture Diagram
- Show how the project application connects to back-end components.

### 5. AWS Database Usage Screenshot
- Include a screenshot proving your AWS Database usage.
- Examples: Vercel Storage Configuration, AWS Console showing your Aurora/DynamoDB resource, or similar.

---

## Optional — Bonus Points

Publish a piece of content (blog, podcast, video) covering how you built your project using one of the specified AWS Databases and Vercel.

- You can submit more than one piece of content.
- Content must be **published publicly** (e.g., builder.aws.com, LinkedIn, medium.com, dev.to, YouTube, etc.) and **not unlisted**.
- Must include language stating you created the piece of content **for the purposes of entering this hackathon**.
- When sharing on social media, use the hashtag **#H0Hackathon**.
- See full rules for details.
