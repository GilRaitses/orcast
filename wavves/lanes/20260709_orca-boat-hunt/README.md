# HUNT: orca-boat-hunt

Bash Hackathon (July 10) build. A player pilots the existing orcast orca over
the real bathymetry with WASD + mouse-look, sinks simple arcade boats by
ramming them, and can sonar-ping nearby targets (boats and named Salish Sea
places) then Floo-teleport straight to a pinged target.

Files in this lane home:

- `waveset.md` - the authority doc: grounding, locked decisions, wave
  structure, acceptance criteria, model routing, escalation catch.
- `dispatch.md` - the fenced paste block sent to the dispatched lane
  orchestrator.
- `findings/` - Wave 1 discovery findings, one file per topic.
- `gate-captures/` - HUNT-INT / HUNT-ACCEPT evidence (screenshots, logs).
- `decisions/` - any decision records written during the run.
- `deliverable/` - the portable Bash.tv build-prompt package (non-code
  deliverable for pasting into the Bash.tv agent prompt box on July 10).

Status: chartered, not yet dispatched. See `wavves/registry.yml` for the
lane's registry entry. HUNT-W1 (discovery) and HUNT-W2 (build) run
un-gated once dispatched; HUNT-INT (integration into one route) and
HUNT-ACCEPT (browser verification) are GATED on O0 approval.
